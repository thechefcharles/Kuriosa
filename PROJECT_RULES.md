# Kuriosa Project Rules

- **TypeScript everywhere** — No JavaScript in source; strict types.
- **No business logic in UI** — Keep components presentational; use hooks, services, route handlers.
- **Modular architecture** — Features in dedicated modules; avoid monolithic files.
- **Phase discipline** — Do not add features outside the active phase scope.
- **Prefer readability over cleverness** — Clear, maintainable code first.
